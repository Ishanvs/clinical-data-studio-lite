# Architecture Notes

## Objective
Provide a lightweight prototype of a clinical data review platform.

## Current Flow
Clinical dataset CSV
→ Upload through UI
→ Preview and review metrics
→ Flag rows with missing values

## Future State
Clinical sources
→ Ingestion layer
→ Standardization
→ Validation
→ Anomaly detection
→ Review dashboard
→ Audit and traceability layer

## Why this matters
Clinical data is often fragmented across multiple systems and vendors. A unified review layer reduces manual effort and helps teams detect issues earlier.
